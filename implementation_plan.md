# 셸리(Life Gallery) 앱 상세 구현 설계도 (Implementation Plan)

본 설계도는 `mission.md`와 `rules.md`에 정의된 시니어 전용 '인생 스토리' 창작 및 무대 관리 앱의 기능 명세와 개발 순서를 정의합니다. 사용성(접근성)과 단계별 AI 개입(에이전트 선택 시스템)을 중심으로 작성되었습니다.

## 1. 아키텍처 및 핵심 원칙
- **프레임워크**: Next.js (App Router), React, TypeScript
- **상태 관리**: React State 단위 관리 (단계별 Step 기반 Linear 흐름 적용)
- **접근성 원칙**: 대형 텍스트, 아이콘 기반 직관적 내비게이션, 복잡성 배제 (하나의 화면엔 하나의 역할만).
- **데이터 저장**: Supabase (임시 저장 및 최종 메타데이터) 및 로컬 PC 다운로드 시스템('나의 인생 갤러리' 폴더).

---

## 2. 단계별 UI 및 데이터 플로우 설계

### 1단계: 사진 엔진 (Photo Engine)
- **UI 목표**: 사진 선택과 AI 리터칭의 매끄러운 연계
- **구성 요소**:
  - 사진 업로드 컴포넌트 (`<내 사진첩 열기>`)
  - **AI 에이전트 선택 카드 UI**: `<더 멋있게 만들기>` 클릭 시 등장
    - `[회화 스타일로]`, `[선명하게]`, `[콜라주 생성]` 3가지 대형 카드 제공
  - **요구사항 입력창**: 에이전트 선택 후, "어떻게 바꿔드릴까요?" 프롬프트 입력란 제공 (예: "수채화처럼 따뜻하게 해줘")
  - **내비게이션 하단바**: `<이전>`, `<이 사진으로 할래요>`, `<다음 글쓰기>` 

### 2단계: 글쓰기 엔진 (Writing Engine)
- **UI 목표**: 글쓰기에 부담을 느끼는 시니어와, 직접 쓰고 싶은 시니어를 위한 이원화 모드 지원
- **구성 요소**:
  - **이원화 모드 선택**: `<내가 쓸게요>` (에디터 열림) / `<AI에게 부탁하기>` (사진 기반 프롬프트 엔진 가동)
  - **직접 쓰기 에디터**: 폰트 종류, 크기 조절, 색상 변경 툴바 제공
  - **AI 창작 모드**: 로딩 스피너와 함께 "사진의 감성을 읽어내고 있습니다" 문구 제공 후 결과물 오버레이.
  - **내비게이션 하단바**: `<됐어요>` (수정 완료), `<사진과 글이 잘 어울려요>`, `<다음 음악 만들기>`

### 3단계: 음악 엔진 (Music Engine)
- **UI 목표**: 분위기를 결정하는 BGM을 쉽게 입히거나 AI로 생성
- **구성 요소**:
  - **모드 선택**: `<선택하여 넣기>` (기본 BGM 리스트 제공) / `<AI에 작곡 부탁하기>`
  - 리스트 UI: 카드형 플레이버튼 (미리듣기 기능 탑재)
  - AI 작곡: 앞서 생성한 텍스트/사진 무드를 분석하여 Generative Music 요청 (Mocking 가능)
  - **내비게이션 하단바**: `<이전>`, `<이 음악으로 할래요>`, `<다음 목소리>`

### 4단계: 목소리 엔진 (Voice Engine)
- **UI 목표**: 오디오 녹음의 두려움을 덜고 마법 같은 AI 변조 기능 경험 제공
- **구성 요소**:
  - 녹음 컨트롤러: `<녹음 시작>`, `<듣기 (반복)>`
  - **AI 목소리 변환 (필터)**: `<다른 목소리로>` (일반, 시 낭송 톤, 웅장한 톤 등으로 변조)
  - **저장 시스템**: 브라우저 API를 활용해 최종 결과물(합성된 영상/오디오)을 로컬 PC의 `[나의 인생 갤러리]` 폴더로 다운로드.
  - **내비게이션 하단바**: `<이전>`, `<발표 준비>`

### 5단계: 무대 매니저 (Stage Manager)
- **UI 목표**: 오프라인 발표회 혹은 온라인 SNS 공유를 위한 완벽한 매니지먼트 환경 구축
- **구성 요소**:
  - **공유 카드 UI**: 카카오톡, 페이스북, 밴드 등 대형 버튼
  - **구글 주소록 연동 (MCP 고려)**: Google Contacts API 연동을 통해 내 연락처 목록을 조회 후 초대장 전송 대상 체크박스 리스트업
  - **4대 체크리스트**: 
    1. `[발표일정]` 일정 및 장소
    2. `[무대준비]` 기기 점검 / 사진 현상
    3. `[사람/업체]` 초대 명단 / 도우미 확인
    4. `[의상/화장]` 단정한 준비 상태
  - 각 항목을 터치해 체크할 수 있는 프로그레스(달성률) 인디케이터 제공.

## 3. 실행 단계 가이드 (Execution Roadmap)

이 설계도를 바탕으로 아래 순서에 맞추어 **정확히 "한 단계씩"** 코딩을 요청하시면 됩니다.

- [ ] **Phase 1**: 1단계 '사진 엔진' 전면 재개발 (AI 에이전트 선택 UI 및 입력창)
- [ ] **Phase 2**: 2단계 '글쓰기 엔진'의 이원화 UI (직접 쓰기 / AI 창작)
- [ ] **Phase 3**: 3단계 '음악 엔진' 개발
- [ ] **Phase 4**: 4단계 '목소리 엔진' 및 로컬 폴더 저장 시스템
- [ ] **Phase 5**: 5단계 '무대 매니저' (SNS + 연락처 연동 + 체크리스트)
